# 📅 Day 22 - Striver SDE Sheet Challenge

## ✅ Problems Solved

### 1. Kth Largest Element in an Array (LeetCode)

- **Approach:** Min Heap (Priority Queue)
- **Time Complexity:** O(N log K)
- **Space Complexity:** O(K)

**Key Learning:**
- Learned how to maintain only the K largest elements using a Min Heap.
- Understood why a heap of size K is more efficient than sorting the entire array.
- Strengthened understanding of Priority Queue applications.

---

### 2. Max Heap Implementation (GFG)

- **Approach:** Array-Based Heap
- **Time Complexity:**
  - Push → O(log N)
  - Pop → O(log N)
  - Peek → O(1)
  - Size → O(1)
- **Space Complexity:** O(N)

**Key Learning:**
- Implemented Heap from scratch using an array.
- Practiced Heapify-Up and Heapify-Down operations.
- Improved understanding of internal heap structure.

---

### 3. Maximum Sum Combination (GFG)

- **Approach:** Max Heap + Set
- **Time Complexity:** O(K log K)
- **Space Complexity:** O(K)

**Key Learning:**
- Learned how to generate only the required maximum combinations instead of all possible pairs.
- Used a Max Heap to always retrieve the largest remaining sum.
- Used a set to prevent duplicate index pairs.
