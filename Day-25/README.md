### 1. Next Greater Element I (LeetCode)

- **Approach:** Monotonic Stack + HashMap
- **Time Complexity:** O(N + M)
- **Space Complexity:** O(M)

**Key Learning:**
- Learned how to use a monotonic decreasing stack to efficiently find the next greater element.
- Used a hash map to store precomputed answers for constant-time lookup.
- Strengthened understanding of monotonic stack applications.

### 2. Sort a Stack (GFG)

#### Approach 1: Copy + Sort

- **Time Complexity:** O(N log N)
- **Space Complexity:** O(N)

**Key Learning:**
- Converted the stack into a vector, sorted it using STL, and rebuilt the stack.
- Practiced using STL algorithms to simplify implementation.

#### Approach 2: Auxiliary Stack

- **Time Complexity:** O(N²)
- **Space Complexity:** O(N)

**Key Learning:**
- Used an auxiliary stack to maintain elements in sorted order.
- Learned insertion-sort-like behavior using only stack operations.
- Improved understanding of stack manipulation techniques.

### 3. Valid Parentheses (LeetCode)

- **Approach:** Stack
- **Time Complexity:** O(N)
- **Space Complexity:** O(N)

**Key Learning:**

- Detected invalid sequences immediately when mismatches occurred.
- Reinforced stack usage for parsing and expression validation.
