## ✅ Problems Solved

### 1. Palindrome Linked List (LeetCode 234)

- **Approach:** Vector Comparison & In-place Reversal
- **Time Complexity:** O(N)
- **Space Complexity:**
  - Vector: O(N)
  - In-place Reversal: O(1)

**Key Learning:**
- Solved the problem first using extra space with a vector.
- Optimized it by reversing the second half of the linked list.
- Revised the Slow & Fast Pointer technique and linked list reversal together.

---

### 2. Linked List Cycle II (LeetCode 142)

- **Approach:** Hash Set & Floyd's Cycle Detection
- **Time Complexity:** O(N)
- **Space Complexity:**
  - Hash Set: O(N)
  - Floyd's Algorithm: O(1)

**Key Learning:**
- Learned how Floyd's Algorithm not only detects a cycle but also finds its starting node.
- Understood the mathematical intuition behind resetting one pointer to the head after the first meeting point.
- Compared both brute-force and optimized approaches.

---

### 3. Flattening a Linked List (GFG)

- **Approach:** Recursion + Merge Two Sorted Linked Lists
- **Time Complexity:** O(N × N × M)
- **Space Complexity:** O(N)

**Key Learning:**
- Applied recursion to flatten the remaining list first.
- Reused the merge technique from Merge Two Sorted Lists.
- Improved confidence in solving recursive linked list problems.
